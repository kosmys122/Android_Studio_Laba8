# The Finals Catalog — лабораторная работа №9

**Приложение для Android** с прокручиваемым списком, созданное с использованием **Jetpack Compose**.  
Показывает каталог популярных аниме с изображениями, названиями и описаниями.

---

##  Какие элементы использованы

###  UI-компоненты Jetpack Compose
- `LazyColumn` — вертикальный прокручиваемый список с ленивой загрузкой
- `Card` — карточка для каждого элемента списка
- `Image` — отображение обложки аниме
- `Text` — название и описание
- `Modifier` — управление отступами, размерами и расположением
- `Column` — вертикальное размещение текста внутри карточки
- `ContentScale.Crop` — обрезка изображений под заданные размеры

###  Архитектурные и логические элементы
- `data class Final` — модель данных с аннотациями:
  - `@StringRes` для заголовка
  - `@StringRes` для описания
  - `@DrawableRes` для изображения
- Класс `Datasource` — источник данных:
  - возвращает `List<Final>` из 10 элементов
- `MainActivity` — точка входа
- `@Preview` — быстрый просмотр UI без запуска эмулятора

###  Ресурсы
- `res/drawable/` — 30 изображений (JPG/PNG)
- `res/values/strings.xml` — названия и описания скина
- `res/mipmap-*` + `ic_launcher.xml` — адаптивная иконка

##  Какая логика добавлена

### 1. Логика работы со списком
- Данные полностью отделены от UI (через `Datasource`)
- `FinalCatalogApp()` загружает список один раз при запуске
- `LazyColumn` автоматически:
  - переиспользует элементы при прокрутке
  - создаёт только видимые карточки
- Каждая карточка получает:
  - ресурс названия
  - ресурс описания
  - ресурс изображения

### 2. Логика отображения
- Изображение растягивается по ширине и обрезается под `450.dp`
- Название — крупный стиль (`headlineSmall`)
- Описание — стандартный стиль (`bodyMedium`)
- Карточки отделены друг от друга отступом `8.dp`

### 3. Логика иконки (Adaptive Icon)
- Создан **фон** (background) и **передний план** (foreground)
- Использован **Image Asset Studio** для генерации иконок под все плотности
- Основной элемент помещён в **Safe Zone (≈66px)**, чтобы не обрезался

### 4. Логика локализации
- Все тексты хранятся в `strings.xml`
- Поддержка многоязычности без изменения кода
- Названия и описания передаются через `stringResource()`

##  Как запустить проект

1. Открыть проект в **Android Studio**
2. Выбрать эмулятор или реальное устройство
3. Нажать `Run` (зелёная стрелка)
